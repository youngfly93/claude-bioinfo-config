# 清除 AI 痕迹

扫描并清除交付文件中的 AI 使用痕迹。

> 完整交付场景优先用 `bio-deliver`。本 skill 是 AI 痕迹专项清理，可被 `bio-deliver` 编排，也可在只需要扫描/清理 AI 痕迹时单独使用。

## 单一真源

关键词定义统一在脚本，**不要在别处重复维护**：
`${CLAUDE_PLUGIN_ROOT:-$HOME/.claude}/skills/bio-deliver/scripts/ai_trace_scan.py`

- **HARD**：明确 AI 标识（工具名 Claude/ChatGPT/GPT…、第一人称自指、`# Generated by` 等代码注释、Office 元数据）→ 可自动清除。
- **SOFT**：AI 高频套话（综上所述/值得注意的是/In conclusion…），但也可能是正常学术用语 → 只标记 `action=review`，绝不自动删。

## 流程

### 1. 扫描
```bash
python3 ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude}/skills/bio-deliver/scripts/ai_trace_scan.py scan <目标目录>
```
输出 JSON，每项含 `severity`(hard/soft) 与 `action`(auto/review)；退出码 1=有发现。覆盖文本/代码文件、Office 元数据、docx/pptx/xlsx 正文 xml。

### 2. 自动清除 HARD
```bash
python3 ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude}/skills/bio-deliver/scripts/ai_trace_scan.py clean <目标目录>
```
清除文本/代码文件的 HARD 命中 + Office 元数据。重要文件清除前先 `git status` 确认可回滚（脚本对 Office 用临时副本，文本类直接改写）。

### 3. 人工复核 review 项（SOFT + Office 正文）
脚本不会自动改这些，避免误删正文：
- **SOFT 套话**：判断是不是 AI 腔。是 → 改写成自然学术表达；正常用语 → 留着。
- **docx/pptx/xlsx 正文命中**：用 document-skills:docx 等打开手动改，保持实质内容不变。

### 4. 重扫确认
重跑 `scan`，确认 `action=auto` 清零、`review` 已处理。

## 注意
- 只改 AI 痕迹，不动文档实质内容。
- Office 元数据被清后脚本已重新打包，无需手动重打。
