# 项目清爽收口

agentic 分析既写一堆 md（审计/说明/各版本草稿→多版本并存、矛盾），也在项目里撒一地系统/临时/缓存垃圾（`._*`、`~$锁`、`__pycache__`、`*.log`、`.RData`…）。本 skill 两条道把项目**收口成清爽树**：

- **(A) md 收口** → 单一真源 + `DOCS_INDEX.md`，归档旧版进**一个** `_archive/`（下方工作流 1–6）。
- **(B) 垃圾清扫** → 无歧义系统/临时/缓存文件扫进**一个** `.work/_swept_<日期>/`，保持项目树清爽（下方「垃圾清扫」节）。

> 原则见全局/工作区 CLAUDE.md「文档卫生」：一事一文件、原地改、版本交 git。本 skill 治理**存量**散落与垃圾。

## 硬边界（先记住）

- **只归档，不删除**：md 旧版 `mv` 进**一个** `_archive/`、垃圾 `mv` 进**一个** `.work/_swept_<日期>/`，一律**保留可恢复**，绝不 `rm`（用户另说"确认删"才删）。
- **保护的是「真实文件」，不是「目录」**：判垃圾**按文件模式、不按所在目录**——`plan.md`/`spec.md`、`results/`/`figures/`/`scripts/`/`reports/`/`delivery/` 里的**真实**数据/脚本/报告、`numeric_reference.tsv`/`report_claims.tsv`/`execution_log.md`/`HANDOFF.md`/`audit/` 这些**不匹配垃圾模式**的文件一律原样不动。但**垃圾模式文件（`._*`/`~$*`/`*.log`/`__pycache__`…）无论落在哪个目录都算垃圾、照扫**——包括 `results/._foo.tsv`、`reports/._报告.docx` 这类藏在真源夹里的 `._` sidecar（它们是真实文件的 macOS 附属垃圾，不是真源本身）。`.git/`、`_archive/`、`.work/` 整个 prune 不进扫。
- **只扫无歧义垃圾**：有歧义的中间物（`.rds`、`results/` 外散落的 `.tsv/.csv/.png`、非编号脚本）**只标记给用户看、不移**——它们可能是下游要读的，误伤代价高（本仓一堆 `_v2`/`no_` 按模式动文件的自反误报疤）。
- **dry-run 先行**：先列"将移动什么→移去哪"给用户，**确认了才动**。
- **只在单个项目目录内**：绝不跨顶层合作者目录（全局纪律：别动别人的目录）。动手前 `git status` + 确认范围。
- **真源拿不准就问**：哪个 md 是当前权威、某文件算不算垃圾有疑问 → 列出来问用户，不擅自判定。

## 工作流

### 1. 限定范围
确认要收口的**单个项目目录**（默认当前目录，但不上溯到合作者根）。

### 2. 只读扫描
```bash
# 列 md + 标多版本/副本命名（按目录看散落程度）
find <项目> -name '*.md' ! -path '*/.git/*' ! -path '*/_archive/*' ! -name '._*' 2>/dev/null
# 多版本/副本命名命中
find <项目> -name '*.md' ! -path '*/.git/*' ! -name '._*' 2>/dev/null \
  | grep -iE '(_v[0-9]|_final|_最终|_new|_新|副本| copy|\([0-9]\)| [0-9]\.md|backup|bak|old)'
```
- 按**主题/文件名词干**聚类（如所有 `AUDIT_*`、所有 `*plan*`、所有 `*methods*`）。
- 每组按修改时间 + 是否 git 跟踪 + 内容完整度排序。

### 3. 判定当前真源
每组挑"当前权威"，依据（按优先级）：
- 文件名最干净（无 `_v*/_final/副本` 后缀）且 git 跟踪的；
- 内容最新、最全；与 `plan.md`/`HANDOFF.md` 指向一致的。
- 其余归为「旧版 / 过程稿」。**拿不准 → 列出问用户**。

### 4. 给方案、等确认
输出收口方案表，确认后才动：

```markdown
## 文档收口方案（<项目>）
| 主题 | 当前真源(保留) | 拟归档进 _archive/ | 依据 |
|------|---------------|-------------------|------|
| 审计 | audit/AUDIT.md | AUDIT_*_FINAL.md ×3、v1.2.* ×4 | 后者为历史快照 |
```

### 5. 执行（确认后）
- 旧版/过程稿 `mv` 进项目根**一个** `_archive/`（保留，不删）。
- 真源留原位、文件名规范化（去掉 `_final` 等后缀时用 `git mv`）。

### 6. 产出真源清单 DOCS_INDEX.md
在项目根写/更新 `DOCS_INDEX.md`，让接手者一眼知道哪几个权威：

```markdown
# DOCS_INDEX · <项目>  (更新: <绝对日期>)
## 权威文档（认这些）
- plan.md — 设计与进度
- HANDOFF.md — 当前状态快照
- reports/<最终报告>.docx — 交付报告
- audit/AUDIT.md — 最新审计
## 历史（_archive/，仅备查，勿据此分析）
- 旧版审计 / 各版本草稿 …
```

## 垃圾清扫（B 道：无歧义系统/临时/缓存 → 一个 .work/_swept）

保持项目树清爽——把散一地的垃圾归拢进一个可恢复的角落，不误伤真源。

### B1. 只读扫描 + 分三类
```bash
# 无歧义垃圾（可扫）：系统/临时/缓存/锁/编译产物
find <项目> \( -path '*/.git/*' -o -path '*/_archive/*' -o -path '*/.work/*' \) -prune -o \
  \( -name '._*' -o -name '~$*' -o -name '.DS_Store' -o -name 'Thumbs.db' \
     -o -name '*.log' -o -name '*.tmp' -o -name '*.pyc' -o -name '.Rhistory' \
     -o -name '.RData' -o -name '*.swp' -o -name '*~' -o -name '__pycache__' \) -print 2>/dev/null
```
- **可扫（无歧义）**：上面命中的系统/临时/缓存/锁/编译文件 → 拟移进 `.work/_swept_<日期>/`。
- **只标记不移（有歧义）**：`*.rds`、`results/` 外的散落 `.tsv/.csv/.png`、非编号脚本、看不懂用途的大文件——**列出来问用户**，别自作主张移（可能是下游画图/报告要读的）。
- **绝不碰**：硬边界里那批真源/数据/记录。

### B2. dry-run 方案表，等确认
```markdown
## 垃圾清扫方案（<项目>）
| 类 | 处置 | 文件（相对路径） |
|----|------|-----------------|
| 无歧义垃圾 | → `.work/_swept_20260708/`（保留可恢复） | `._plan.md`、`scripts/__pycache__/`、`run.log` … |
| 有歧义(只标记) | **留原位，请你判断** | `results/tmp_matrix.rds`、`fig_draft.png` … |
```

### B3. 执行（确认后）
- `mkdir -p .work/_swept_<日期>/`，把无歧义垃圾 `mv` 进去、**保留相对路径**，写一份 `_swept_manifest.tsv`（原路径 → 新路径）便于恢复。
- 只 `mv` 不 `rm`；`.work/` 本就可随时整体删（清爽不留痕）。
- 有歧义项一个不动，等用户逐项发话。

## 纪律
- 只 `mv` 不 `rm`；md 归档只用一个 `_archive/`、垃圾只用一个 `.work/_swept_<日期>/`，别再造多个。
- 不跨合作者目录；不动 `results/`/`figures/`/`scripts/` 等**真源/数据**（A 道只收 md，B 道只扫无歧义垃圾，有歧义只标记）。
- dry-run 先列给用户、`git status` 先看；移动文件是危险操作，宁保守勿误伤。
- 收口后提醒：以后**原地改真源**、可再生的边干边扔 `.work/`、新版交 git，别再堆文件名（`canonical_guard` 会拦多版本命名）。
