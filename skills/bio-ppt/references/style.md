# Shirley Wu 风格设计原则（提取自她 5 份公开讲稿）

写内容时遵守这些原则，PPT 才"像她"。规格/字段见 [content-spec.md](content-spec.md)。

## 一句话
**极简白底 + 数据即故事**：克制到几乎没有装饰，用大留白、清晰层级、一抹标志红制造高级感。

## 七条铁律
1. **一页一个意思**。一个观点撑不满一页？正好——留白就是设计。宁可拆成 3 页。
2. **字越少越好**。标题 ≤ 6 词；正文一页 ≤ 3 短句 / 列表 ≤ 5 条。她最狠的页只有一句话。
3. **层级靠"字重+字距+颜色+大小写"，不靠堆字号**。英文小标题一律大写 + 大字距。
4. **90% 黑白灰 + 一个标志红**。红只给真正的重点（编号、序号、链接、关键词）。永不满屏配色。
5. **深底用于"呼吸点"**：章节分隔、金句、情绪转折用 `dark:true`，与白底页交替制造节奏。
6. **编号章节弧线**：用 `section` + `side_label`（`01. question` / `02. method` …）串起叙事，像写书分章。
7. **数据即故事**：每页先问"想让观众感受/记住什么"，再决定放什么。结尾常落在"data → story"。

## 叙事骨架（推荐顺序）
cover → section(00/01) → statement(抛问题) → content/two_column/list(方法与结果) →
photo(关键图表) → statement(金句收束) → closing

她的经典编号弧线可借鉴：`identity → spark → dream → burnout → transition → rekindling`，
迁移到科研即：`question → method → data → results → takeaway`。

## 双语混排
- 英文术语/标题 + 中文说明并置（如 `WORKFLOW 分析流程`、`in-house 自产数据`）。
- 脚本已自动让英文走拉丁字体、中文走 CJK 字体，无需手动干预。

## 不要做（这版风格的边界）
- 不用手绘涂鸦、彩铅渐变、emoji 满天飞——本 skill 走她"极简专业"那一面（用户已选定）。
- 不堆满、不加边框装饰线、不用多种强调色。
- **正式 SCI 图表稿 / 临床报告 / 客户交付别用此风格**：它偏演讲/分享气质，正式交付仍用规范模板。
  适用场景：组会、paper 分享、project pitch、学术报告、个人演讲。
