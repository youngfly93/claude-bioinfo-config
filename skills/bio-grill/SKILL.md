# 生信开工前审问

跑分析之前，把分析设计逐条对齐到没有歧义，再开跑。生信第一大坑是"代理跑了一个你并不完全想要的分析"——这个 skill 就是在开跑前堵住它。

<what-to-do>

围绕这次分析的设计审问用户，直到双方对"要跑什么、怎么跑、用什么参数"达成共识。沿着分析决策树逐个分支走，一次只问一个问题，等用户回答再问下一个。

**每个问题都附带你的推荐答案**（给依据，不是甩选项）。

能从 `plan.md` / `CONTEXT.md` / 现有脚本 / 数据本身查到的，**先查再问，不要问用户已经写明的东西**。

</what-to-do>

<supporting-info>

## 模式分叉

开场先用一句话判定本次属于哪种，不确定就直接问用户一句"这次是快速探索还是高风险/交付级？"

- **探索模式（默认）**：只问 4 条最核心的——①样本分组/配对 ②contrast 方向（比哪两组、谁减谁）③批次是否处理 ④关键阈值（padj/FC）。其余用常规默认值**直接给出建议**，不逐条盘。适合"先看一眼趋势"。
- **高风险 / 交付 / 临床相关模式**：按 [CHECKLIST.md](./CHECKLIST.md) 逐轴狠盘，每条都要用户确认。适合交付级、面向临床报告（如肠癌358、proteomicAnalysis 交付）、要过审稿/客户的分析。错了代价大，所以宁可多问。（注意：这是"分析设计审问"，不替代任何临床决策。）

## 审问纪律

- **一次一问**：问完一个，等回答，再问下一个。不要一口气列十个问题。
- **带推荐答案**：每个问题给出"我建议 X，因为 Y"，让用户确认或改，而不是逼用户从零决策。
- **先查再问**：能从 `plan.md`（步骤与进度）、`CONTEXT.md`（术语与样本命名）、`scripts/`（已有代码）、或数据本身（维度、列名、分组列）查到答案的，自己去查，别问用户。
- **依据可查文献**：关键设计决策（阈值、方法、参考基因组/注释版本"业界怎么定"）自己拿不准时，用 `nature-academic-search` 查文献（PubMed/CrossRef），让推荐答案附上文献依据，不只凭先验。高风险/交付/临床相关模式尤其值得查。
- **当场指出冲突**：用户的回答和 `plan.md`、现有脚本、或前面已确认的决策矛盾时，立刻挑出来。"你刚说按 padj<0.05，但 scripts/02_deg.R 里写的是 0.1——以哪个为准？"
- **不替用户编事实**：不知道的样本来源、参考版本、分组含义就问，不要假定。

## 流程

1. **读项目上下文**：先看 `plan.md`、`CONTEXT.md`（若有）、`scripts/` 现有脚本，建立对这次分析的初步理解。
2. **判定模式**：探索 or 交付（见上）。
3. **逐轴审问**：按 [CHECKLIST.md](./CHECKLIST.md) 的通用骨架顺序走，并叠加对应数据类型（RNA-seq / 蛋白组 / 单细胞 / 16S）的专属追问。探索模式只取核心 4 条（样本分组/配对、contrast 方向、批次、关键阈值）。
4. **决策落档（稀疏）**：审问中确认的"为什么这样分析"决策，**仅在满足触发线时**提议落档到 `docs/analysis-decisions/`，格式与触发线见 [DECISION-FORMAT.md](./DECISION-FORMAT.md)。其余的不记。
5. **输出设计确认单**：审问结束，输出下面这张表，用户点头后才开跑。

## 输出：分析设计确认单

审问收尾时，把对齐结果汇总成一张表，让用户最后一次过目：

| 维度 | 本次确认 |
|------|----------|
| **分析目标** | 这次要回答的生物学问题 + 最终产物（图/表/报告） |
| **样本与分组** | 样本数、分组、配对/独立、重复、协变量 |
| **数据与参考** | 数据类型、来源、参考基因组/注释版本、count vs TPM |
| **QC 与过滤** | 低质量/低表达过滤标准、离群样本、缺失值处理 |
| **批次处理** | 有无批次、如何发现、处理方式（协变量/ComBat/不处理） |
| **归一化与转换** | 方法（TMM/vst/rlog/log2/quantile…） |
| **比较（contrast）** | 到底比哪两组、方向、是否多对比 |
| **统计模型与工具** | 工具（DESeq2/limma/edgeR/wilcox…）+ 设计矩阵 + 选择依据 |
| **阈值与多重检验** | padj 方法与阈值、FC 阈值、显著性定义 |
| **下游** | 富集（数据库/背景基因集）、可视化、ML（若有） |
| **可复现产物** | 随机种子、环境记录、中间对象保存、脚本编号、输出路径 |
| **待落档决策** | 满足触发线、需写入 docs/analysis-decisions/ 的条目（若有） |

确认无误 → 开跑。如有未决项，标出来再问。

</supporting-info>
