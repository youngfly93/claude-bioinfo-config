# 分析决策落档格式

审问中确认的"为什么这样分析"决策，**仅在满足下面触发线时**才落档。其余的别记，否则变成没人看的噪音。

## 三层分工（别混在一起）

| 文件 | 放什么 | 不放什么 |
|------|--------|----------|
| `CONTEXT.md`（项目根） | 术语、样本命名规则、分组/contrast 的含义、项目实体 | 执行步骤、参数、决策理由 |
| `plan.md` | 当前分析步骤、进度、待办 | 术语定义、需长期溯源的决策理由 |
| `docs/analysis-decisions/` | **"为什么这样分析"** 的决策与理由 | 术语、进度 |

## 何时该记（任一条成立即可）

1. **有人可能合理反对**——存在真实的替代方案而你选了其一。
   - ✅ 算：DESeq2 vs edgeR vs limma-voom；保留 vs 去除某批次；ANCOM-BC vs LEfSe。
   - ❌ 不算：`padj<0.05`、BH 校正这类行业默认——没人会问。
2. **改回代价大**——选错了要重跑一大串。
   - ✅ 算：参考基因组 build（hg19 vs GRCh38）、样本分组定义、配对 vs 独立设计。
3. **审稿人 / 客户大概率会问**。
   - ✅ 算："为什么去掉/保留这个批次？""为什么排除这 N 个样本？""为什么用这个背景基因集？"

三条都不沾的，扔进 `plan.md` 一句话带过即可。

## 位置与编号

- 目录：`docs/analysis-decisions/`，**懒创建**——第一条决策出现时再建。
- 文件名：顺序编号 `0001-slug.md`、`0002-slug.md`……扫描已有最大编号 +1。

## 模板（一段话即可）

```md
# {决策的简短标题}

{1–3 句：背景是什么、决定了什么、为什么。}
```

例：

```md
# 0003-保留测序批次作为协变量而非 ComBat 校正

两批 RNA-seq（2023Q1 / 2023Q4）PCA 上按批次分离，但批次与分组无混杂。
决定在 DESeq2 设计矩阵中加入 batch 协变量（~ batch + group），保留原始 count，
不对 count 做 ComBat，以免在小样本下抹掉真实分组信号。
```

## 可选字段（多数决策不需要）

仅在确实有价值时加：
- **Status**（proposed / accepted / superseded by 00NN）——决策日后被推翻时有用。
- **被否方案**——当"为什么没选 X"值得记住、否则半年后又有人提议时。
- **后果**——有非显而易见的下游影响需要提醒时。
