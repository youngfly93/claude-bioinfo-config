# 审问清单

按下面的**通用骨架**顺序逐轴走，再叠加本次**数据类型专属**的追问。每条都给"该问的点 + 一个常规默认推荐"——推荐是用来让用户确认或推翻的起点，不是定论。

探索模式只取标 ⭐ 的核心 4 条（样本分组/配对、contrast 方向、批次、关键阈值），其余给默认建议直接跳过。

---

## 通用骨架（所有数据类型共有）

### 0. 目标与产物
- 这次分析要回答的**生物学问题**是什么？一句话说清。
- 最终**产物**是什么——探索性图表、可发表主图、还是交付报告？产物决定了审问火力。

### 1. 样本与设计 ⭐
- 每组多少样本？分几组？组名是什么？
- **配对还是独立**？（同一病人治疗前后 = 配对；不同病人 = 独立）
- 生物学重复 vs 技术重复，分别多少？
- 有没有需要纳入模型的**协变量**（性别、年龄、批次、测序深度）？
- *默认*：≥3 生物学重复/组；独立设计；协变量先看与分组是否混杂再决定。

### 2. 数据与来源
- 数据类型？来源（GEO GSE* / TCGA / DepMap / 自测）？
- **参考基因组与注释版本**（GRCh38 vs hg19；GENCODE 版本）——下游基因 ID 映射全靠它。
- 拿到的是 **raw count、TPM/FPKM、还是已归一化矩阵**？这决定能用哪些方法。
- ID 类型（Ensembl/Symbol/Entrez）与去重策略。
- *默认*：明确记录 build 与注释版本；count→DESeq2/edgeR，连续值→limma。

### 3. QC 与过滤
- 低质量样本怎么判定、是否剔除？
- 低表达/低丰度特征怎么过滤（如 CPM>1 在 ≥N 个样本中）？
- 缺失值情况与处理。
- *默认*：先做样本聚类/PCA 看离群；过滤掉在所有样本几乎不表达的特征。

### 4. 批次 ⭐
- 有没有批次效应（不同测序批、不同平台、不同时间）？
- **怎么发现的**——PCA/聚类按批次而非分组分开？
- 处理方式：**放进模型当协变量** / **ComBat 等显式校正** / **不处理**？（批次与分组混杂时尤其要小心，校正可能抹掉真信号）
- *默认*：能放协变量就放协变量（保留原始 count 给 DESeq2/limma），不轻易对 count 做 ComBat。

### 5. 归一化与转换
- 用什么归一化/转换方法？为什么？
- *默认*：count→DESeq2 内置/edgeR TMM；可视化/聚类用 vst 或 rlog；连续强度→log2 + 适当归一化。

### 6. 比较（contrast）⭐
- **到底比哪两组**？方向是谁比谁（treat vs control，别反）？
- 是否多个 contrast？多组两两比还是 vs 共同对照？
- 交互项需要吗？
- *默认*：明确写出 `groupB - groupA`，逐一确认方向；多对比时统一对照组。

### 7. 统计模型与工具
- 用哪个工具，为什么（见数据类型专属节）？
- **设计矩阵**长什么样（`~ batch + group`）？
- *默认*：根据数据类型选（见下），设计矩阵把协变量放在分组前。

### 8. 阈值与多重检验 ⭐
- 多重检验校正方法（BH/BY/Bonferroni）？
- **padj 阈值**（0.05 还是 0.1）？**FC/logFC 阈值**？
- "显著"的完整定义（padj < X 且 |log2FC| > Y）。
- *默认*：BH 校正，padj<0.05，|log2FC|>1（探索可放宽到 0.1 / 0.585）。

### 9. 下游
- 富集分析：用哪些库（GO/KEGG/Reactome/MSigDB）？**背景基因集**是什么（全基因组 vs 表达基因）？
- 可视化清单（火山图、热图、PCA、通路图…）。
- 机器学习/建模（若有）：算法、特征、训练/验证拆分、防过拟合。
- *默认*：背景用"检测到表达的基因"而非全基因组；图遵循 SCI 标准（白底、无网格线、舒适配色）。

### 10. 可复现产物
- **随机种子**固定了吗（涉及聚类、采样、ML 时）？
- 环境/版本记录（sessionInfo / conda env / renv）？
- 中间对象（RDS/RData）保存策略与脚本编号约定？
- 输出路径与命名。
- *默认*：固定 seed；保存 sessionInfo；脚本按 `01_`、`02_` 编号；结果进 `results/`。

## 数据类型专属追问

### A. RNA-seq / 差异表达
- 工具选择依据：**DESeq2 vs limma-voom vs edgeR**——有 raw count 且样本中等规模常用 DESeq2；大样本/复杂设计 limma-voom 更快更灵活。为什么选这个？
- 输入是 count 还是 TPM？（TPM 不该直接喂 DESeq2）
- vst 还是 rlog 做可视化转换（样本多用 vst）？
- 低表达过滤的具体规则？
- 独立过滤 / cooks 距离离群处理用默认吗？
- padj 与 FC 阈值（见通用 8）。

### B. 蛋白组 / DIA-PTM
- 上游：**DIANN 还是 pQuant**？拿到的是 protein-level 还是 precursor/peptide-level？
- **缺失值机制**（MNAR 低丰度缺失 vs MAR）与填补策略（不填/最小值/KNN/MICE）——蛋白组缺失值处理直接影响结论。
- log2 转换 + 归一化方法（median / quantile / VSN）？
- 差异检验用 limma（蛋白组常用）还是 t/wilcox？
- **PTM 专属**：是否 site-level 分析？定位置信度阈值（localization probability，如 >0.75）？是否对蛋白总量做归一化以分离"修饰变化"与"丰度变化"？

### C. 单细胞
- QC 阈值：**nFeature / nCount / 线粒体比例 mt%** 的上下限怎么定（看分布而非套默认）？
- 双细胞（doublet）检测与去除（DoubletFinder/scrublet）？
- 归一化：**LogNormalize 还是 SCTransform**？
- HVG 数量、PCA 用多少主成分？
- **聚类分辨率** resolution 取多少？是否扫多个值？
- 多样本/多批次**整合方式**：Harmony / CCA / RPCA / 不整合？依据是什么？
- 细胞类型**注释依据**：marker 基因来源、自动注释工具、人工校验？

### D. 16S / 宏基因组
- 特征表来源（QIIME2/DADA2 ASV vs OTU；宏基因组 MetaPhlAn/Kraken）？
- **丰度过滤 / 流行度过滤**（prevalence，如出现在 ≥10% 样本）规则？
- **稀释（rarefaction）还是不稀释**？α 多样性常需统一深度，β 多样性更推荐 rarefy 或用组成性方法——怎么权衡？
- α 多样性指标（Shannon/Simpson/observed）；β 多样性**距离度量**（Bray-Curtis / UniFrac weighted/unweighted）？
- 差异丰度方法：**ANCOM-BC / LEfSe / DESeq2 / ALDEx2**——选哪个？（注意微生物数据的**组成性 compositional** 问题，普通方法易假阳性）
- 测序深度差异是否作为协变量？
